# 47_Embedded_Product_UX_AntiPatterns

> Version: 1.0
> Status: Strategic

---

# 목적

본 문서는 Embedded UI/UX 안티패턴을 정의한다.

---

# 1. 핵심 철학

```text
나쁜 UX는 사용자를 피곤하게 만든다.
```

---

# 2. 대표적인 안티패턴

| 안티패턴 | 문제 |
|---|---|
| 메뉴 Depth 과다 | 길 잃음 |
| 작은 숫자 다량 표시 | 피로 |
| RGB Flash | 저급감 |
| 과한 Animation | 산만 |
| 상태 혼합 표시 | 혼란 |

---

# 3. Embedded UI 흔한 실수

```text
❌ 디버그 정보 남발
❌ 개발자 중심 UI
❌ 기능 우선 UI
```

---

# 4. 좋은 UX 특징

```text
단순함
안정감
예측 가능성
```

---

# 5. 최종 원칙

```text
사용자는 시스템 내부를 알고 싶어하지 않는다.
```
