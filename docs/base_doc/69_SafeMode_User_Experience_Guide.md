# 69_SafeMode_User_Experience_Guide

> Version: 1.0
> Status: UX Strategy

---

# 목적

본 문서는 SafeMode UX 전략을 정의한다.

---

# 1. 핵심 철학

```text
SafeMode는 사용자를 보호하기 위한 것이다.
```

---

# 2. 표시 목표

사용자가:

```text
현재 위험 상태임을 즉시 이해
```

해야 한다.

---

# 3. 표시 내용

| 내용 | 설명 |
|---|---|
| SAFE MODE | 현재 상태 |
| 원인 | Sensor Fail |
| 조치 | CHECK SENSOR |

---

# 4. 금지

```text
❌ 개발자용 에러 코드만 표시
❌ 복잡한 stack trace
```

---

# 5. UX 원칙

```text
불안감보다
명확함이 중요하다.
```

---

# 6. 최종 원칙

```text
사용자는 지금 무엇을 해야 하는지 이해 가능해야 한다.
```
