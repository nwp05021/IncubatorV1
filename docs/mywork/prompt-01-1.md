## 1.5 MENU 3 — 수동 작동 테스트 (ManualControl)

**목적**: 부화 자동 제어를 **일시 정지**하고 릴레이/팬을 수동으로 직접 ON/OFF.  
실물 하드웨어 설치 후 배선 점검 목적.

**⚠️ 핵심 안전 규칙**:

진입 시: ManualControlEntered Event 발행
         ClimateControlModule은 이 Event 수신 시 제어 루프 일시 정지
         (SafeMode는 아님 — Relay 자체는 살아 있음)

종료 시: ManualControlExited Event 발행
         ClimateControlModule 제어 루프 재개
         모든 수동 상태 OFF (ManualTestController.releaseAll())

# 2. 단일진실 원칙 중요

프리셑 적용 방법:

- 시스템은 닭/오리/거위 ... 등에 대한 제어 로직을 내부적으로 가지고 있다.
- 사용자가 프리셑 메뉴에서 닭을 선택하면
    1. 프리셑 로직에 의해 IncubationPlan 테이블을 일차 별로 생성
    2. 이 생성된 테이블이 단일 진실이다.
    3. 이 후 모든 로직은 이 테이블을 기준으로 움직인다.
    4. 공장초기화 기본 값으로 IncubationPlan 은 닭의 데이터로 만들어진다.

# 3. WiFi 가 끊기더라도 로컬 단독으로 완벽하게 작동하도록 설계
- 내부 TimeManager 에 의해 년월일시분초 데이터를 관리
- WiFi 가 연결되면 일정 주기별로 서버와 시간 동기화
- WiFi 가 끊기더라도 마지막 저장된 일시 정보를 기준으로 작동해야 한다.

일단 이 정도의 원칙으로 로직을 완성해 주기를 바란다.
